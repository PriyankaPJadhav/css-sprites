Css sprites

What is a Sprite

An image sprite is a collection of images put into a single image at defined X and Y coordinates.

To display a single image from the combined image, you could use the CSS background-position property, defining the exact position of the image to be displayed.

A web page with many images, particularly many small images, such as icons, buttons, etc. can take a long time to load and generates multiple server requests.

Using the image sprites instead of separate images will significantly reduce the number of HTTP requests a browser makes to the server, which can be very effective for improving the loading time of web pages and overall site performance.

Simple Example

Sprite image

We could calculate position like this 
	.code-icon:hover {
	  background-image: url('sprite.png');
	  background-position: -216px -72px;
	  width: 72px;
	  height: 72px;
	}
  	
	.list-icon{
		width: 72px;
		height:72px;
		display: block;
		cursor: pointer;
		background: url('sprite.png') no-repeat left top;
	}

	.icon-2{
		background-position: -72px 0;
	}

	.icon-1:hover{
		background-position: 0px -72px;
	}

	.icon-2:hover{
		background-position: -72px -72px;
	}

In Scss, we could write the following styles for the code icon hover: 
		$icon-width: 72px;
		$icon-height: 72px;
		$icons: url('sprite.png');

		.code-icon:hover {
		  background-image: $icons;
		  background-position: -($icon-width * 3) -($icon-height * 1);
		  width: $icon-width;
		  height: $icon-height;
		}

Advantages

  1. Fewer HTTP requests, which means less server overhead.
  2. Fewer HTTP requests also means reduced page load time on browser.
  3. Images appear faster (Especially the hover state images) because the Image Sprite is loaded once and for all.

Disadvantages

  1. More development time for slicing images, combining them, and programming background positions in the CSS.
  2. More maintenance time. Whenever a modification is needed, the whole sprite needs to be re-generated carefully
  3. Less SEO friendly. Some images are better be placed in the HTML rather than being backgrounds. HTML images can contain Titles and Alternative texts that are more beneficial to SEO, while CSS backgrounds cannot.
